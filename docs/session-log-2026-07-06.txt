KEY DECISIONS & FINDINGS (the things to remember)
--------------------------------------------------------------------------------
- GENERATOR, NOT TRANSLATOR. Grow outward from verified gold; never
  "type a sentence in".
- Agent C: two comparisons (score vs ORIGINAL, repair vs CURRENT drift);
  keep C deterministic — the combo matcher (AUC 0.993) is the single
  source of sound truth everywhere.
- Morphology is a lookup, not a skill: LemmInflect / suffix-rules +
  Lexique validation before any GPU. This alone ~5x'd the inventory.
- Stricter seed (0.70) + unchanged output floors (0.60/0.45): drift
  control without discarding data.
- Homonym classes already existed in the repo (707 EN / 4,583 FR /
  33,660 Lexique) — wired in as the ladder's homonym bucket.
- espeak-ng phonemizes whole French phrases WITH liaison ("les amis" ->
  le-z ami), so phrase-level filtering judges junctures for free;
  sandhi_fr.py provides the same phonology symbolically.
- Placement: weave = stage-3 generator; prosody = stage-4 judge channel;
  sandhi between generation and judging.
- Flywheel measurably compounds (cycle-2 finds descend from cycle-1
  parents), and diminishing candidate counts per cycle are the healthy
  "vein mining out" signal to move to the learned expansion.
- Known loose ends on the LOCAL machine: gpu_char_model.py generate()
  bug (c/ch collision) on phrase-weave branch; unresolved merge conflict
  in the local repo; vast.ai instance possibly still running; 52MB
  homophone_model_gpu.pt not yet uploaded anywhere durable.
