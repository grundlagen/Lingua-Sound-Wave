# Dictionary v7 — best of v5 + v6

`build_v7.py` unions v5's retrieval with v6's single-phoneme decoder, the fix
diagnosed in `WHY_V6_FEWER.md`. v5's method is the better base (exhaustive,
rule-rich, multiple candidates, less corner-cutting); v6's decoder adds reach the
retrieval blocking misses plus the filler/line carves it cannot make.

## What v7 does

- **RETRIEVAL leg (v5):** block the whole French lexicon, rank by the rule-rich
  combo, keep **top-K per word** (multiple candidates, like v5). **Allophone-aware
  blocking:** the bigram shortlist key is unioned over each French word's
  `matcher._variants` (schwa-drop, nasal-split, rhotic ʁ~ɹ, diphthong-smooth), so
  allophonic realizations are reachable — and combo ranks with the same variants.
  Allophones enter as *match-time rules on both sides*, the accent-independent way
  (`REPRESENTATION.md`), not as stored data.
- **DECODER leg (v6):** the single-phoneme poetry trie, run when retrieval is weak
  (< A) and for filler/line carves.
- Keep the better per `(en, fr)` by combo; v5 re-mining gates set the `gold` flag.

## Result (1,200 EN × 6,000 FR sample, 60 s)

```
2069 entries (retrieve 1693 + decode 376)   tiers: 92 S / 282 A / 1695 B
gold 92   filler 24   decode-only additions 376
```

**1.7 entries/word** vs v6's ~0.46 — the multiple-candidate retrieval base alone
triples the yield. And the decoder leg is genuinely complementary: it recovered
**S-tier homophones retrieval missed** —

```
more -> maures   get -> guette   she -> chie   read -> ride
police -> pelisse   red -> raid
```

— words whose best French match was outside retrieval's bigram-Dice shortlist or
top-N lexicon, but which the decoder carved cleanly. Neither leg alone finds all
of these; the union does.

## Integration

`dictionary-v7-integrated.json` = full v5 ∪ v7's new entries (1,644 new → 13,432).
Use this for Round Rabbit and the engine: v5's accumulated coverage + v7's
allophone-aware retrieval and decoder recoveries.

## Scaling

The sample used 1,200 EN × 6,000 FR. Full scale (12,000 EN × 25,000 FR, v5's run
size) is linear and reproduces v5's coverage *plus* the decoder recoveries and
filler carves — i.e. v7 ≥ v5 on words, with the line material on top. Gold tier
uses the June-11 augment gates; recompute `learn_costs.py` on the integrated
certified set before the next pass to tighten the matcher further.

One-line: v7 = v5 retrieval (multiple, allophone-aware) ∪ v6 single-phoneme
decoder, arbiter-ranked — the best of both, with the granularity you wanted.
