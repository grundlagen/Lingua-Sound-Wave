# Dictionary v6: v5's ethos + one-segment fillers + the session's fixes

`build_v6.py` re-mines in v5's spirit (real lexicon → decode → matcher-rank →
tier, composition-ready) with four changes earned this session. This run is a
**450-word seed** (the pipeline scales with `--limit`); numbers are from it.

## What v6 changes vs v5

1. **One-segment fillers admitted.** The poetry trie lets van Rooten's filler
   words (un /œ̃/, et /e/, on /ɔ̃/, a, aux, y, eau, le, de…) emit at length 1;
   content words still need ≥2 segments. These are the "extra words" v5's
   `MIN_WORD_SEGS=2` structurally excluded.
2. **Arbiter ranking.** Candidates are ranked by the **coverage-aware matcher
   combo**, not the decoder's coverage-blind similarity — the fix for the
   Humpty/épidémie bug, now applied to *every* mined entry.
3. **Coherence field.** Each entry carries a bigram-LM fluency score on the FR
   side, so composition can prefer carves that read as French.
4. **Re-mine, not re-label.** Words v5 couldn't carve because they needed a
   filler now get entries (the honest-negative lesson).

## Seed-run results (450 frequent EN words)

| | |
|---|---|
| entries | 261 (58% yield) |
| tiers | 27 S, 50 A, 184 B |
| novel vs v5 | 13 |
| use a filler | 9 (3%) |

S-tier looks like v5's gold but now coherence-scored (get→guette, she→chie,
two→tout coh 0.63, said→cède). New **filler-carry** entries v5 could not produce:
`only → on lie`, `anyone → on n` (coh 0.89), `data → d est`, `yourself → et elfes`.

## The honest finding: fillers are composition primitives, not word entries

Only **3%** of single-word entries use a filler — because a short English word
usually maps to one French content word; it rarely *needs* a filler. Fillers earn
their keep in **multi-word / line** carving (Humpty Dumpty → un petit d'un petit),
which is the generation engine's job, not the word table's. So v6's filler value
is mostly the **`v6-fillers.tsv` primitives layer** — the building blocks the
engine inserts when re-cutting a stream — plus the handful of word-level
filler-carries. The improvements that touch *all* entries are #2 (arbiter
ranking) and #3 (coherence), not the filler count.

## What v6 is, precisely

- a re-mined, **coverage-aware-ranked**, **coherence-tagged** word table in v5's
  schema (`dictionary-v6.tsv`), plus
- the **filler primitive layer** (`v6-fillers.tsv`) the generation engine draws on.

It does not replace v5 as a word-for-word table (v5/reranked is already
near-ceiling there); it adds the scaffolding and the corrected ranking that the
*generation* path needs. The next step pairs it with whole-line coverage-aware
search (one carve across a whole line, not per-window), where the fillers and the
coherence field do their real work.

## Scaling

The seed used `--limit 450` (~26 s). The same pipeline over the full frequent-word
list (or v5's 7,286 headwords) produces the complete v6; runtime is linear. Gating
quality remains the L2-coherence model — the coherence column is bigram-grade and
should be recomputed when a real L2 model lands.
